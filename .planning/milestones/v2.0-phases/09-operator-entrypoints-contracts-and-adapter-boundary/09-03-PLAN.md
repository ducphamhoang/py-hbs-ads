    - Load registry and resolve sender by platform identity; add `canonical_person_key` to the event when resolved.
    - Load open prompts from prompt store and call `match_reply`.
    - If no match, return envelope with `ok=False`, `action_taken="clarification_needed"`, `requires_clarification=True`, `clarification_reason`, and candidate data.

    - If safe, call `notion_adapter.apply_actions(..., execute=execute)`.
    - Return envelope with `ok=True`, `action_taken="dry_run_planned"` in dry-run or `"write_applied"` in execute, `write_applied` matching live write status, prompt/match/update fields, audit event names, and nested data for resolve/match/plan/apply details.
